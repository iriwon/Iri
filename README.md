# 🎮 WISP-ER: Afterimage - 게임 설명서 & 기획서

> **"죽음은 끝이 아니라, 새로운 발판이 된다."**

## 1. 프로젝트 개요 (Project Overview)

*   **프로젝트명**: WISP-ER: Afterimage
*   **장르**: 2D 횡스크롤 퍼즐 플랫폼 (2D Puzzle Platformer)
*   **플랫폼**: WebGL / Windows PC
*   **입력 방식**: 키보드 & 마우스
*   **개발 엔진**: Unity 2022+

## 2. 게임 컨셉 & 목표 (Core Concept)

### 👻 핵심 메카닉: 잔상 (Afterimage)
이 게임의 가장 독창적인 요소는 **'죽음을 이용한 퍼즐 해결'**입니다. 플레이어가 함정에 닿거나 높은 곳에서 떨어져 사망하면, 그 자리(또는 특정 시점)에 **'잔상(Ghost)'**이 생성됩니다.
부활한 플레이어는 자신이 남긴 **잔상을 밟고 더 높은 곳으로 점프하거나, 발판으로 삼아** 갈 수 없었던 곳을 돌파해야 합니다.

*   **협력 아닌 협력**: 과거의 나(잔상)와 현재의 내가 협력하여 스테이지를 클리어합니다.
*   **전략적 죽음**: 단순히 죽지 않으려고 피하는 것이 아니라, **"어디서 죽어야 이 잔상이 도움이 될까?"**를 고민해야 합니다.

## 3. 상세 시스템 설명 (Game Systems)

### A. 유령(잔상) 시스템
*   **생성 조건**:
    1.  **사망 시**: 플레이어가 함정(가시 등)에 닿거나 낙사하면, 사망 직전(또는 점프 정점)의 위치에 잔상이 남습니다.
    2.  **수동 생성**: 특정 키(`Shift`)를 눌러 현재 위치에 즉시 잔상을 생성할 수 있습니다.
*   **최대 개수**: 화면(맵)에 최대 **3개**의 잔상만 존재할 수 있습니다. 4번째 잔상이 생성되면 가장 오래된 잔상이 사라집니다.
*   **상호작용**: 잔상은 물리적인 충돌체(Collider)를 가지며, 플레이어는 이를 밟고 설 수 있습니다 (플랫폼 역할).

### B. 잔상 관리
*   **수동 제거**: 잔상을 잘못 만들었을 경우, `Left Alt` 키를 눌러 가장 오래된 잔상을 지울 수 있습니다.
*   **초기화**: 스테이지를 클리어하고 다음 스테이지로 넘어가면 모든 잔상은 자동으로 소멸합니다.

### C. 퍼즐 기믹 (Gimmicks)
*   **열쇠 & 문**: 맵에 흩어진 열쇠를 모두 모아야 잠긴 문이 열리며 다음 구역으로 이동할 수 있습니다. (자동 ID 매칭 시스템 적용)
*   **중력 반전**: 특정 아이템/구역을 통과하면 중력이 뒤집혀 천장을 바닥처럼 걸을 수 있습니다.
*   **투명 발판**: 평소엔 보이지 않지만 잔상을 이용해 위치를 가늠해야 하는 플랫폼이 존재합니다.

### D. 진행 구조
*   **월드 맵 구조**: 스테이지 1-1부터 1-9까지 끊김 없이 이어지는 심리스(Seamless) 구조를 채택했습니다.
*   **체크포인트**: 주요 구간마다 체크포인트(깃발 등)가 있어 사망 시 해당 지점에서 빠르게 부활합니다.
*   **타이머**: 우측 상단에 월드 전체 클리어 시간을 측정하는 타이머가 표시됩니다.

## 4. 조작법 (Controls)

| 동작 | 키 입력 | 설명 |
| :--- | :---: | :--- |
| **이동** | `W`, `A`, `S`, `D` / `방향키` | 좌우 이동 및 상하(사다리 등) |
| **점프** | `Space` | 점프 (잔상 위에서도 가능) |
| **잔상 생성** | `Left Shift` | 현재 위치에 즉시 잔상 생성 (체력 소모 없음) |
| **잔상 제거** | `Left Alt` | 가장 오래된 잔상 1개 제거 |
| **재시작** | `R` | 끼임 발생 시 현재 스테이지 재시작 |
| **일시정지** | `ESC` | 일시정지 메뉴 호출 (계속하기 / 타이틀로) |

## 5. 인터페이스 (UI)

*   **위습(Wisp) 카운터**: 플레이어 주변을 공전하는 구체(Orb) 3개는 현재 사용 가능한(또는 남은) 잔상의 개수를 시각적으로 보여줍니다. 잔상이 생성되면 구체가 비활성화되고, 제거되면 다시 켜집니다.
*   **스테이지 알림**: 스테이지 진입 시 화면 중앙에 스테이지 이름(예: "1-1")이 우아하게 표시됩니다.
*   **타이머 UI**: 우측 상단에 `분:초.밀리초` 단위로 현재 플레이 시간을 실시간 표시합니다.

## 6. 개발 노트 (Technical Notes)

*   **Seamless Stage System**: 별도의 로딩 화면 없이 카메라 이동과 페이드 효과만으로 스테이지가 자연스럽게 전환되도록 구현되었습니다.
*   **Smart Key System**: 열쇠와 문의 연결이 ID 기반으로 자동화되어 있어, 레벨 디자인 수정 시 연결 오류를 최소화했습니다.
*   **Feedback**: 사망 시 화면 흔들림(Shake), 시간 정지(Hit Stop), 화면 왜곡(Aberration) 등의 연출(Juice)을 통해 타격감을 극대화했습니다.
